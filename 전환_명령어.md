# [Claude Code 전달용] 전 화면 물가 판단 기준 전환 지시서

## 배경

지금까지 "이번 달이 비싼 달인지 싼 달인지"를 판단하는 로직 일부가 **월별 등락률(전월비 %, 계절평균 시트)** 을 기준으로 되어 있는데, 이는 "가격이 얼마나 크게 뛰는 달인지"를 말해줄 뿐 "실제로 어느 달이 비싼지"는 알려주지 못한다. (예: 배추는 8월에 전월비 등락률이 가장 크지만(+60.1%), 실제 지수 레벨이 가장 높은 달은 9월이다 — 8월 상승분이 9월까지 누적되기 때문.)

따라서 **"몇 월이 비싸고 싼지"를 다루는 모든 화면 요소는 등락률이 아니라 "연평균 대비 레벨 편차"로 통일**한다.

## 새 기준 계산식

```
데이터 소스: 생산자물가지수_품목별_월별등락분석.xlsx → "원자료(월별지수)" 시트만 사용
             ("계절평균(월별등락률)" 시트는 이 계산에 사용하지 않는다)

1. 품목별로 원자료의 65개월(2021.01~2026.05) 지수값을 달력 월(1~12월)별로 그룹핑
2. 그룹별 평균 = 해당 월의 평균 지수 레벨 (예: 배추의 모든 9월 지수값 평균)
3. 연평균 = 12개 달력 월 평균의 평균 (동일 가중치, 관측치 수로 가중하지 않음)
4. 월별 편차(%) = (해당 월 평균 지수 / 연평균 − 1) × 100
   → 결측월은 계산에서 제외, 최소 관측치 미달 시 "데이터 부족" 처리
```

이 12개 값(1~12월 편차%)이 이제 모든 화면에서 공통으로 쓰는 **단일 소스**다. 화면마다 새로 계산하지 말고, 공통 유틸 함수(`getMonthlyLevelDeviation(itemId): number[12]`) 하나로 만들어 재사용할 것.

## 신호등/문구 표현 규칙 (공통)

- ❌ 쓰지 말 것: "많이 오르는 달", "많이 내리는 달", "등락률이 가장 큰 달"
- ✅ 쓸 것: "평균보다 비싼 시기", "평균보다 싼 시기", "가장 비싼 달", "가장 싼 달"
- 신호등 판정(🟢/🟡/🔴)은 위 12개 편차값 중 이번 달 값의 백분위로 계산 (하위 33%=🟢, 중간=🟡, 상위 33%=🔴) — 메인 화면에 이미 반영된 로직과 동일한 함수 재사용

## 화면별 수정 범위

### 1. 홈 화면

| 요소 | 처리 |
|---|---|
| **이번 달 최다 상승/하락 TOP5** | **그대로 둔다.** 이건 "이번 달이 지난달보다 실제로 얼마나 변했는지"를 보여주는 실시간 등락률이 맞는 개념이라, 레벨 편차로 바꾸면 안 된다. 데이터 소스도 그대로 `월별증감률(전월비)` 시트의 최신 월 값 유지 |
| **찜 품목 이번 달 통계** ("평년 대비 O% 비싸요/싸요") | **레벨 편차로 전환.** "평년 대비"라는 표현 자체가 계절 수준 비교이므로, `getMonthlyLevelDeviation`의 이번 달 값을 그대로 사용 |
| **계절 저렴 품목 카드뉴스** ("배추는 보통 O월이 가장 저렴해요") | **레벨 편차로 전환.** 지금은 계절평균 시트의 "가장많이내리는달"을 쓰고 있는데, 이걸 `getMonthlyLevelDeviation`에서 최솟값을 갖는 달로 교체. 문구도 "저렴해져요"(등락 표현) → "저렴해요"(수준 표현)로 다듬기 |

### 2. 메인(탐색) 화면

- 이미 반영 완료 (막대그래프, 가장 비싼/싼 달, 신호등 전부 레벨 편차 기준). 다른 화면 작업 시 이 화면의 로직/유틸 함수를 그대로 재사용할 것

### 3. 마이페이지

| 요소 | 처리 |
|---|---|
| **찜한 품목 리스트의 +%/-% 뱃지** | **레벨 편차로 전환.** 현재 이번 달 편차값을 그대로 표시 |
| **장바구니 시뮬레이터** ("과거 최저점 대비 O% 비싼 수준") | **레벨 편차로 전환.** "과거 최저점"은 12개 편차값 중 최솟값(=가장 싼 달)으로 재정의하고, 비교 문구도 "이번 달 vs 그 품목이 보통 가장 쌌던 달" 기준으로 통일 |
| **알림 설정**(전주/전월 대비 급등락 시 알림) | **그대로 둔다.** 알림은 "갑자기 튀었는지"를 감지하는 실시간 등락률 개념이 맞으므로 유지 |

## 작업 순서 제안

1. `getMonthlyLevelDeviation(itemId)` 공통 유틸을 원자료 시트 파싱 로직 위에 구현 (없다면 메인 화면 구현 코드에서 로직 추출해 공용화)
2. 홈 화면의 "찜 품목 통계", "계절 카드뉴스"를 이 유틸로 교체
3. 마이페이지의 "찜 목록 뱃지", "장바구니 시뮬레이터"를 이 유틸로 교체
4. TOP5(홈)와 알림(마이페이지)은 손대지 말 것 — 실시간 등락률 유지가 맞는 케이스
5. 전 화면에서 "오르는/내리는 달" 표현을 "비싼/싼 시기" 표현으로 일괄 검색 후 수정
