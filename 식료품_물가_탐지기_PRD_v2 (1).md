# 식료품 물가 측정 탐지기 + 구매 타이밍 도우미 PRD

**버전**: v2.0 (Phase 1 데이터 소스 변경)
**작성일**: 2026-07-16
**상태**: Draft
**변경 요약**: Phase 1 데이터 소스를 한국소비자원 생필품가격정보 API → **생산자물가지수 품목별 월별등락분석(xlsx)** 로 대체. 이에 따라 계절 카드뉴스 기능이 Phase 1로 앞당겨지고, 신호등 판정 로직과 화면별 기능 범위가 조정됨.

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

소비자는 식료품 가격이 "지금 비싼 건지 싼 건지"를 직관적으로 판단하기 어렵다. 개별 마트 전단지나 뉴스 기사로는 특정 품목의 계절성·연간 변동폭을 파악할 수 없고, "지금 사도 되는 타이밍인지"에 대한 근거 있는 답을 주는 서비스가 없다.

본 서비스는 공공·통계 가격 데이터를 기반으로,
1. 이번 달 물가 변동 상위 품목을 한눈에 보여주고
2. 사용자가 찜한 품목의 "계절/연간 통계 대비 현재 위치"를 계산해주며
3. 품목별 구매 타이밍을 신호등(초록/노랑/빨강)으로 직관적으로 안내하는

식료품 물가 측정 및 구매 타이밍 도우미 서비스다.

### 1.2 타겟 사용자

- 장보기 지출을 관리하고 싶은 1인 가구 / 가계 관리자
- "지금 사도 되는지"를 매번 검색하기 귀찮은 실사용자
- 가격 변동에 민감한 특정 품목(김치 재료, 육류, 수산물 등)을 정기 구매하는 사용자

### 1.3 핵심 가치 제안

| 문제 | 해결 |
|---|---|
| 지금 가격이 비싼지 싼지 감이 안 잡힘 | 과거 지수 분포 대비 백분위 기반 신호등 제공 |
| 계절별 저렴한 시기를 모름 | 계절평균 데이터 기반 카드뉴스 (**Phase 1부터 제공**) |
| 관심 품목 가격을 매번 찾아봐야 함 | 찜 + 알림 |
| 이번 달 vs 다음 달 중 언제 사는 게 이득인지 모름 | 장바구니 시뮬레이터 (지수 기반 상대 비교) |

## 2. 데이터 소스 전략

### 2.1 Phase 1 (MVP) — 생산자물가지수 품목별 월별등락분석 (xlsx)

**선정 이유**: 65개월(2021.01~2026.05)치 62개 품목의 월별 지수와, 동일 월끼리 평균낸 **계절 패턴(어느 달에 오르고 내리는지)이 이미 계산되어 있는** 데이터. 별도의 계절성 분석 로직을 자체 구현하지 않아도 되어 MVP 개발 속도에 유리.

**파일 구성**

| 시트명 | 내용 |
|---|---|
| 안내 | 데이터 설명, 카테고리 구성, 해석 유의사항 |
| 원자료(월별지수) | 품목별 65개월 생산자물가지수 원자료 |
| 월별증감률(전월비) | 전월 대비 등락률(%) — 수식(`=이번달/전달-1`)으로 산출되어 있어 원자료 갱신 시 자동 재계산 |
| 계절평균(월별등락률) | 2021~2026년 동일 월의 전월비 등락률 평균 + **가장많이오르는달/평균상승률/가장많이내리는달/평균하락률** 컬럼 |
| 차트(카테고리별 계절패턴) | 9개 카테고리별 월평균 등락률 선그래프 |

**커버리지**

- **9개 카테고리 · 62개 품목**: 곡물류(쌀,보리쌀,콩) / 서류(감자,고구마) / 채소류(엽경채·기타, 배추·무·양파·마늘 등 다수) / 과실류(사과,배,포도 등) / 축산물(쇠고기,돼지고기,닭고기,달걀 등) / 임산물 / 수산물(생물) / 수산물(가공)
- 실제 그로서리 앱에서 다룰 법한 핵심 품목 대부분을 이미 포함 (배추, 돼지고기, 계란, 사과, 고등어 등)

**데이터 특성 및 제약 (설계에 반드시 반영)**

| 특성 | 설명 | 설계 영향 |
|---|---|---|
| **지수(index) 데이터, 실가격(원) 아님** | 대부분 `2020=100` 기준 (단, **양배추만 `2022.12=100`** — 기준 시점 상이) | 신호등·통계는 "몇 % 비싸졌다/저렴하다" 상대 비교로만 표현. "얼마인지" 원화 표시는 불가 → 장바구니 시뮬레이터는 절대 금액이 아닌 **지수 기준 상대 등락률**로 안내 |
| **월별 데이터** (주간/일간 아님) | 65개월 시계열 | 그래프는 **월별 라인 차트**로 설계 (Phase 1 초안의 "그래프" UI를 주간이 아닌 월간 기준으로 확정) |
| **생산자(도매) 단계 가격** | 소매가 아닌 생산자물가지수 | 사용자 체감 소매가와 괴리가 있을 수 있음 — UI에 "도매 시세 기준" 임을 명시 필요 |
| **가중치(weight) 컬럼 존재** | 품목별 비중값 포함 | 카테고리 전체 등락률 등 추후 가중 평균 산출 시 활용 가능 (Phase 1 필수는 아님) |
| **일부 품목 결측(NaN) 구간** | 예: 양배추는 2023년 이전 데이터 없음 | 백분위/계절평균 계산 시 결측 구간 제외 처리 필요 |
| **매장/지역 정보 없음** | 판매점별·지역별 비교 불가 | "판매점 정보 조회" 관련 기능은 **Phase 1 범위에서 제외** (이전 버전 PRD의 마트 비교 기능은 보류) |

### 2.2 Phase 2 — KAMIS 및/또는 실거래가 데이터 연동 (보강)

- Phase 1(생산자물가지수)은 **추세·계절성 판단**에는 강하지만 "지금 마트에서 실제 얼마에 살 수 있는지"는 알려주지 못함
- Phase 2에서 KAMIS(소비자가/도매가, 원 단위) 또는 유사 소매가 데이터를 연동해 **실제 원화 가격 표시, 지역별/매장별 비교** 기능을 보강
- Phase 1의 62개 품목 목록과 Phase 2 품목 코드 간 매핑 테이블 별도 구축 필요 (품목명 기준 매칭, 코드 체계 상이)

### 2.3 데이터 소스 통합 원칙

| 원칙 | 내용 |
|---|---|
| 데이터 유형 태깅 | 모든 내부 상품 레코드에 `value_type` 필드(`index` \| `price_krw`) 필수 부여 — 지수와 실가격을 화면에서 혼동 없이 구분 표시 |
| 기준 시점 정규화 | `양배추`처럼 기준 시점이 다른 품목은 별도 `base_period` 필드로 관리, 타 품목과 지수값 직접 비교 금지 (등락률 비교만 허용) |
| 결측 구간 처리 | 계절평균/신호등 계산 시 결측월 제외, 최소 유효 관측치(예: 해당 월 데이터 3년치 이상) 미달 시 "데이터 부족" 처리 |
| BFF 경유 | Phase 2에서 외부 API 연동 시 인증키는 BFF에서만 관리 (job-seeking 플랫폼과 동일한 보안 패턴) — Phase 1은 정적 파일 기반이므로 별도 인증 불필요, 배치로 파싱해 자체 DB 적재 |

## 3. 화면별 기능 명세

### 3.1 홈 화면

| 기능 | 설명 | 데이터 소스 | 비고 |
|---|---|---|---|
| 이번 달 최다 상승/하락 TOP5 | `월별증감률(전월비)` 시트의 최신 월 값 기준 상위/하위 5개 | Phase 1 | 즉시 구현 가능 |
| 찜 품목 이번 달 통계 | 최근 12개월(또는 전체 65개월) 지수 분포 대비 이번 달 지수의 위치, "평년 대비 O% 비싸요/싸요" | Phase 1 | 도매가 기준임을 라벨링 |
| 계절 저렴 품목 카드뉴스 | `계절평균(월별등락률)` 시트의 "가장많이내리는달" 컬럼 활용 — 예: "배추는 보통 10월에 가장 크게 저렴해져요(평균 -41%)" | **Phase 1로 앞당김** | 62개 품목 전체에 대해 자동 생성 가능, 별도 계절성 분석 로직 불필요 |

### 3.2 메인 화면

- 검색/필터: 카테고리(9종) → 품목(62종) 선택, 찜 등록
- 그래프: **월별 시계열 라인 차트** (X축 = 2021/01~2026/05)
- 구매 타이밍 신호등 + 근거 한 줄

**신호등 판정 로직 (초안, v1 대비 조정)**

```
대상 시계열: 원자료(월별지수)의 해당 품목 65개월 (결측 제외)
percentile = 이번 달 지수가 전체 기간(또는 최근 36개월) 분포에서 차지하는 백분위

if percentile <= 33%:  🟢 지금 사기 좋음   (예: "최근 3년 평균보다 15% 저렴한 도매가 수준이에요")
elif percentile <= 66%: 🟡 보통
else:                    🔴 비쌈           (예: "최근 3년 평균보다 20% 비싼 도매가 수준이에요")
```

- v1과의 차이: 주간(52주) → **월간(36개월 또는 전체 65개월)** 기준으로 변경
- 계절성 보정 옵션(오픈 이슈 1 참고): 단순 절대 지수 백분위 대신, "계절평균 대비 편차"로 볼지는 확정 필요 — 예를 들어 배추는 10월에 원래 저렴해지는 게 정상이므로, 신호등이 항상 "10월=초록"만 반복 표시하는 게 유용한지, 아니면 "계절 패턴을 제거한 후에도 저렴한지"를 봐야 하는지 정책 결정 필요

### 3.3 마이페이지

| 기능 | 설명 | 데이터 영향 |
|---|---|---|
| 로그인/회원가입 | 표준 인증 | 변경 없음 |
| 관심 품목 열람 | 찜 목록 (62개 품목 중 선택) | 변경 없음 |
| 알림 설정 | 지수 기준 급등락(전월 대비 ±X% 이상) 시 알림 | 월별 데이터라 알림 주기는 **월 1회 갱신 기준**으로 확정 (일/주간 실시간 알림 아님) |
| 소비 기록/가계부 | 사용자가 직접 입력한 실제 구매 내역(원화, 수량) | 자체 DB, PPI 데이터와 무관 — 그대로 유지 |
| 장바구니 시뮬레이터 | 이번 달 vs 과거 저점 시점의 **지수 등락률 기반 상대 비교** ("과거 최저점 대비 현재 약 22% 비싼 도매가 수준") | v1의 "총액(원) 비교"에서 **"등락률(%) 비교"로 기능 축소** — 절대 금액 비교는 Phase 2(실가격 데이터) 확보 후 복원 |

## 4. 데이터 모델 개요 (v2)

```
products (내부 마스터)
├─ id
├─ category            (곡물류/서류/채소류(엽경채)/채소류(기타)/과실류/축산물/임산물/수산물(생물)/수산물(가공))
├─ name                (품목명, 예: 배추)
├─ base_period         (예: "2020=100" | "2022.12=100")
├─ weight              (가중치, xlsx 원자료 컬럼)
└─ value_type          (index | price_krw — Phase 2 대비 확장 필드)

index_records
├─ id
├─ product_id (FK)
├─ year_month          (예: 2026-05)
├─ index_value         (원자료 지수값, nullable — 결측 구간 존재)
├─ mom_change_pct      (전월비 등락률, 수식 산출값 그대로 적재)
└─ source              ("생산자물가지수_xlsx_v1")

seasonal_pattern (계절평균 시트 사전 계산값 적재)
├─ product_id (FK)
├─ month               (1~12)
├─ avg_mom_change_pct
├─ top_rise_month
├─ top_rise_avg_pct
├─ top_fall_month
└─ top_fall_avg_pct

user_watchlist / user_purchase_log
└─ (v1과 동일, 자체 관리 데이터로 PPI와 무관)
```

## 5. MVP 범위 (v2)

### 5.1 In Scope (Phase 1)
- 생산자물가지수 xlsx 파싱 → 내부 DB 적재 배치 (원자료 + 계절평균 시트 그대로 활용)
- 홈: TOP5 상승/하락, 찜 품목 통계, **계절 저렴 품목 카드뉴스** (앞당김)
- 메인: 검색/필터, 월별 시계열 그래프, 신호등 판정(지수 백분위 기반)
- 마이페이지: 로그인, 찜, 알림(월 단위), 가계부, 장바구니 시뮬레이터(등락률 기준 상대 비교)

### 5.2 Out of Scope (Phase 1에서 제외, Phase 2 검토)
- 매장/지역별 가격 비교 (데이터 소스에 매장 정보 없음)
- 실제 원화 가격 표시 (지수만 존재)
- 장바구니 시뮬레이터의 절대 금액(원) 비교
- 일/주 단위 실시간 가격 알림 (데이터가 월 단위이므로 근본적으로 불가)

## 6. 오픈 이슈 (희연 확인 필요)

1. **신호등에 계절성을 얼마나 반영할지**: 단순 절대 지수 백분위 vs "계절 패턴 제거 후 편차" — 후자가 더 정교하지만 계산 로직 복잡도 증가
2. **양배추 등 기준 시점 상이 품목 처리**: 다른 품목과 나란히 노출할 때 "기준 연월이 달라 직접 비교 불가"임을 어떻게 안내할지
3. **도매가(생산자물가) 라는 점을 사용자에게 어떻게 전달할지**: 소매가로 오인하지 않도록 UI 문구/디스클레이머 필요
4. **월 단위 갱신 주기가 사용자 기대와 맞는지**: "지금 사기 좋음" 신호등이 한 달에 한 번만 바뀌는 것에 대한 사용자 경험 검토
5. **Phase 2 KAMIS 연동 필요성 재검토**: PPI 데이터가 이미 계절성/트렌드를 상당 부분 커버하므로, Phase 2의 우선순위를 "실가격 확보"에 집중할지 여부
6. **가중치(weight) 컬럼 활용 여부**: 카테고리 전체 등락률을 가중평균으로 보여줄지, 단순 평균으로 충분한지
